BA KeySmith 使用说明
====================

软件名称：BA KeySmith
软件简称：BAKS
当前测试版：v0.2.0-beta.1
作者：HX
适用平台：Windows
适用游戏：蔚蓝档案（Blue Archive）PC 端
项目地址：https://github.com/HXwHSJS/ba-keysmith


一、软件简介
------------

BA KeySmith（简称 BAKS）是由 HX 开发的 Windows 桌面工具，用于蔚蓝档案（Blue Archive）PC 端的键位映射与自定义宏。

它可以把用户实际按下的键转换为游戏内按键输入。

例如：
q -> 1

也就是说，当你按下 q 键时，游戏内会收到 1 键输入。

BA KeySmith 还支持把键盘按键或鼠标按键设置为宏触发键，用来执行连续点击、等待、循环、组合键、鼠标拖拽等操作。


二、v0.2.0-beta.1 测试版更新内容
-------------------------------

1. 支持单个 exe 独立使用。

新版不再要求 BAKeySmith.exe 同目录必须存在 config.json。

2. 配置文件位置变更。

运行时配置会自动保存到：
%APPDATA%\BAKeySmith\config.json

配置内容包括键位映射、宏脚本和开关热键设置。

如果检测到旧版同目录 config.json，会在首次运行时自动复制迁移到用户数据目录。

3. 支持运行中热更新映射。

当映射已经启动时，如果你编辑、新增或删除映射，运行中的按键 hook 会自动重建。

例如：
原本设置 q -> 1，启动映射后将 q 改为 w，新版会自动让 w 生效，并移除旧的 q 触发。

不再需要先停止映射再重新启动。

4. 修复和优化特殊键触发。

增强 ctrl、alt、shift、caps lock 等按键作为触发键时的稳定性，降低点击启动后卡死或特殊键不生效的概率。

5. 优化宏执行。

宏执行速度和按键释放逻辑已优化，减少宏按键与普通 hold 映射互相影响的问题。

6. 宏脚本校验更严格。

如果宏里写了未知键名，会在编译或启动前提示错误，避免“保存成功但运行时没反应”。

7. 补齐更多键名和鼠标按键别名。

例如：
caps lock
insert
delete
home
end
page up
page down
num lock
scroll lock
mouse_right
mouse_x1
mouse_x2

8. 优化编辑体验。

打开宏编辑器、捕获按键窗口等操作时，会暂停当前映射，避免编辑文字时触发游戏输入。


三、快速开始
------------

1. 解压 BA KeySmith 压缩包。

2. 运行 BAKeySmith.exe。

3. 启动蔚蓝档案 PC 端，并确保游戏进程正在运行。

4. 在 BA KeySmith 中添加简单映射或宏映射。

5. 点击“启动映射”。

6. 回到游戏中测试映射效果。

注意：
如果游戏以管理员权限运行，BA KeySmith 也需要以管理员身份运行，否则输入可能无法送达游戏窗口。


四、简单键位映射说明
--------------------

简单映射用于把一个触发键转换成另一个游戏内按键。

支持两种模式：

hold：
按住触发键时持续按住目标键，松开触发键时释放目标键。

tap：
按下触发键时快速点击一次目标键。

示例 1：
触发键：q
目标键：1
模式：tap

效果：
按下 q 时，游戏内触发一次 1 键。

示例 2：
触发键：a
目标键：esc
模式：tap

效果：
按下 a 时，游戏内触发一次 Esc 键。

示例 3：
触发键：ctrl
目标键：1
模式：hold

效果：
按住 ctrl 时，游戏内持续按住 1 键；松开 ctrl 后释放 1 键。


五、宏脚本基础规则
------------------

1. 每条指令占一行。

2. 以 # 开头的内容为注释，不会执行。

3. 指令和参数不区分大小写，但建议使用小写。

4. 参数之间用空格分隔。

5. loop 与 end 必须成对出现，且支持嵌套。

6. wait 后面接等待时间，单位为毫秒。


六、常用宏指令
--------------

press 键名
按下某个键但不释放。

release 键名
释放某个键。

tap 键名
快速点击某个键。

wait 时间
等待指定时间，单位为毫秒。

loop 次数
开始循环。
loop 0 表示无限循环。
loop infinite 也表示无限循环。
loop 5 表示循环 5 次。

end
结束 loop 循环块。

combo 键名1 键名2 ...
触发组合键。
例如 combo ctrl c。

setpos x y
将鼠标移动到屏幕绝对坐标。

setpos x y mouse
将鼠标移动到屏幕比例坐标。
例如 setpos 0.5 0.5 mouse 表示移动到屏幕中心。

setpos_rel dx dy
将鼠标从当前位置相对移动。

drag x y 按键
拖拽到屏幕绝对坐标。

drag x y mouse 按键
拖拽到屏幕比例坐标。

drag_rel dx dy 按键
从当前位置相对拖拽。


七、宏脚本示例
--------------

示例 1：无限循环点击数字键 1，并点击鼠标左键

loop 0
tap 1
tap mouse_left
end

说明：
loop 0 表示无限循环。
tap 1 表示点击一次数字键 1。
tap mouse_left 表示点击一次鼠标左键。
end 表示循环结束位置。


示例 2：鼠标右键触发后循环点击 Esc

loop 0
tap esc
end

说明：
可以将触发键设置为 mouse_right，然后将宏内容设置为以上脚本。
按下鼠标右键后，宏会循环点击 Esc。

注意：
鼠标触发键可以触发宏，但全局鼠标监听通常无法完全阻止原始鼠标点击继续传递给游戏。


示例 3：等待 100 毫秒

wait 100

说明：
wait 后面接等待时间，单位为毫秒。
wait 100 表示等待 100 毫秒。


示例 4：组合键 Ctrl + C

combo ctrl c

说明：
combo 用于触发组合键。


示例 5：鼠标相对拖拽

drag_rel 120 0 left

说明：
drag_rel 表示相对当前位置拖拽。
120 表示向右移动 120 像素。
0 表示纵向不移动。
left 表示使用鼠标左键拖拽。


示例 6：移动到屏幕中心后拖拽

setpos 0.5 0.5 mouse
drag 800 600 left

说明：
setpos 0.5 0.5 mouse 会将鼠标移动到屏幕中心。
drag 800 600 left 会按住鼠标左键拖拽到屏幕坐标 800, 600。

九、配置文件说明
----------------

从 v0.2.0-beta.1 开始，BA KeySmith 的运行配置不再保存到 exe 同目录。

默认配置位置：
%APPDATA%\BAKeySmith\config.json

该文件会记录：
1. 键位映射。
2. 宏脚本。
3. 开关热键设置。

如果你需要备份自己的配置，可以备份以上 config.json 文件。

如果配置出现问题，可以在关闭 BA KeySmith 后删除该 config.json，再重新启动软件。

如果你曾经使用旧版，并且 BAKeySmith.exe 同目录下存在旧 config.json，新版首次运行时会尝试自动复制迁移。


十、常见问题
------------

问题 1：映射或宏没有生效怎么办？

请确认蔚蓝档案 PC 端正在运行，并确认游戏进程名为 BlueArchive.exe。
如果游戏以管理员权限运行，请尝试右键 BAKeySmith.exe，选择“以管理员身份运行”。


问题 2：宏速度能否无限加快？

BA KeySmith 已尽量降低宏执行延迟。
但实际速度仍可能受到 Windows 输入事件调度、电脑性能、游戏自身输入处理速度等因素限制。


问题 3：鼠标右键绑定宏后，原本右键点击还会不会传给游戏？

通常会。
BA KeySmith 可以监听鼠标按键并触发宏，但全局鼠标监听通常无法完全阻止原始鼠标点击继续传递给游戏。


问题 4：编辑宏或捕获按键时为什么不会触发当前映射？

这是为了避免误输入。
当你打开宏编辑窗口、捕获按键窗口或编辑映射时，BA KeySmith 会暂停当前映射，避免你在编辑器里输入 a、q、space 等内容时被转换成游戏按键。


问题 5：运行中编辑映射后，需要停止再启动吗？

不需要。
v0.2.0-beta.1 已支持运行中热更新映射。
编辑、新增、删除映射后，运行中的 hook 会自动更新。


问题 6：为什么 zip 里没有 config.json？

这是正常的。
新版是单 exe 使用模式，配置会在用户数据目录自动生成。


十一、使用注意事项
------------------

1. 本工具检测的目标进程名为 BlueArchive.exe。

2. 如果映射或宏没有生效，请先确认游戏进程正在运行。

3. 如果游戏以管理员权限运行，BA KeySmith 也应以管理员身份运行。

4. 宏速度过快时，可能受到 Windows 或游戏自身处理速度限制。

5. 鼠标触发键可以触发宏，但通常无法阻止原始鼠标点击继续传递给游戏。

6. 请根据自己的设备和游戏环境测试映射效果。

7. 本版本为测试版，如遇到特殊键、宏或热更新异常，建议记录触发键、目标键或宏脚本内容后反馈。
